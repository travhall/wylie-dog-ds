How it works

Five steps from JSON to Figma.

Token Bridge is a thin transform layer. It reads tokens from where they already live, maps them to the Figma Variables API, and preserves the three-tier hierarchy on both sides.

  1. 01 source

    Tokens are defined as JSON.

    Designers and engineers edit the same JSON files in packages/tokens. Style Dictionary compiles them to CSS custom properties; the same JSON is the Token Bridge input.

    packages/tokens/color/interactive.json 
    {
  "interactive": {
    "primary": {
      "value": "oklch(54% 0.18 274)",
      "$type": "color",
      "$description": "Brand primary, buttons & focus rings"
    },
    "primary-hover": { … },
    "primary-active": { … }
  }
}
  2. 02 trigger

    Run pnpm sync:figma.

    Local for one-offs; CI on every merge to main. The first sync from a clean Figma file takes ~6 seconds; incremental syncs run in 2.

    ~/wyliedog/ui — zsh
    ~/wyliedog/ui ❯ pnpm sync:figma
    ⏵ wyliedog-tokens/bridge run
  3. 03 map

    CSS properties → Figma Variables.

    Each tier becomes a Figma Variable Collection. Aliases (semantic referencing primitive) become Figma variable aliases — not flattened copies — so the hierarchy survives the trip.

    P

    Primitives

    color.blue.500 = oklch(54% 0.18 274)

    186
    S

    Semantic → alias to primitive

    interactive.primary = {color.blue.500}

    124
    C

    Components → alias to semantic

    button.primary.bg = {interactive.primary}

    30
  4. 04 modes

    Light and dark, in lockstep.

    Each Figma Variable carries both mode values. Switching the Figma mode swaps every bound layer in one click — the same swap that the .dark class does in code.

  5. 05 propagate

    Figma components update on the spot.

    Every Figma layer bound to a synced variable refreshes the moment the file reopens — no detach, no rebinding, no “please pull” ping. Designers notice the same way engineers do: their work suddenly matches reality.

For designers

The variables panel you already use.

No plugin to install, no proprietary panel. Token Bridge writes into Figma’s native Variables — so designers see token names like interactive/primary, not raw OKLCH strings, in every fill, stroke, and effect picker.

  • Bind any layer property to a token by name.
  • Switch Light ↔ Dark on any frame in one click.
  • Code-set values are read-only in Figma. No accidental drift.

For engineers

One command. No GUI required.

Token Bridge runs from anywhere a pnpm script can: your laptop, CI, a release workflow, a pre-commit hook. The output is diffable, the auth is keychain-managed, and a dry-run flag previews the changes before they hit the file.

install pnpm add -D @wyliedog/token-bridge
init pnpm bridge init
sync pnpm sync:figma
preview pnpm sync:figma --dry-run

Requires Node 20+ and a Figma file with editor access. The bridge talks to Figma’s REST Variables API — no plugin install.

Keep design and code in sync.

Set it up once, forget it forever. Your designers stop asking which value is current, your engineers stop chasing drift, and your tokens become the single thing that’s actually true.
