# Design Token Usage Guide

Our design system uses **Tailwind v4** with **OKLCH colors** for consistent, accessible, and future-proof styling.

## Using Tokens with Tailwind Classes

### Colors
```jsx
// Primary colors
<div className="bg-primary-500 text-white">Primary Button</div>
<div className="bg-primary-100 text-primary-700">Light Primary</div>

// Neutral colors
<div className="bg-neutral-100 text-neutral-900">Secondary Button</div>
<div className="border border-neutral-300">Card Border</div>

// Semantic colors
<div className="bg-success-100 text-success-700">Success Message</div>
<div className="bg-warning-100 text-warning-700">Warning Message</div>
<div className="bg-error-100 text-error-700">Error Message</div>
```

### Spacing
```jsx
// Padding and margins
<div className="p-4 m-2">Standard spacing</div>
<div className="px-6 py-3">Button padding</div>
<div className="space-y-4">Stacked elements</div>

// Gap in flex/grid
<div className="flex gap-3">Flex items</div>
<div className="grid grid-cols-3 gap-6">Grid items</div>
```

### Shadows
```jsx
// Elevation levels
<div className="shadow-sm">Subtle elevation</div>
<div className="shadow-md">Card elevation</div>
<div className="shadow-lg">Modal elevation</div>
<div className="shadow-xl">High elevation</div>
```

## Using Tokens in Custom CSS

When you need to access the raw CSS custom properties:

```css
.custom-component {
  background-color: var(--primary-500);
  border-color: var(--neutral-300);
  padding: var(--spacing-4);
  box-shadow: var(--shadow-md);
}
```

## OKLCH Color Benefits

Our color system uses **OKLCH** (Oklch Lightness Chroma Hue) for superior color science:

- **Perceptual uniformity**: Equal numeric changes create equal visual changes
- **Better accessibility**: More predictable contrast ratios
- **Wider color gamut**: Support for P3 displays and future color spaces
- **Human-readable**: Intuitive lightness, chroma, and hue values

## Browser Support

All OKLCH colors automatically include hex fallbacks for older browsers:

```css
/* Automatically generated by Tailwind v4 */
.bg-primary-500 {
  background-color: #0ea5e9; /* fallback for older browsers */
  background-color: oklch(0.661 0.241 266.19); /* modern browsers */
}
```

## Component Usage Patterns

### Buttons
```jsx
// Using our Button component
import { Button } from '@wyliedog/ui/button';

<Button variant="primary">Primary Action</Button>
<Button variant="secondary">Secondary Action</Button>
<Button variant="outline">Outline Action</Button>
```

### Status Messages
```jsx
// Success state
<div className="bg-success-100 border border-success-500 text-success-700 p-4 rounded">
  Operation completed successfully
</div>

// Warning state
<div className="bg-warning-100 border border-warning-500 text-warning-700 p-4 rounded">
  Please review your inputs
</div>

// Error state
<div className="bg-error-100 border border-error-500 text-error-700 p-4 rounded">
  Something went wrong
</div>
```

### Cards and Containers
```jsx
<div className="bg-white border border-neutral-200 rounded-lg shadow-md p-6">
  <h3 className="text-neutral-900 font-semibold mb-4">Card Title</h3>
  <p className="text-neutral-600 mb-6">Card content with proper spacing.</p>
  <Button>Action</Button>
</div>
```

## Token Categories

### Color Tokens
- **Primary**: `primary-{50-900}` - Brand colors for actions and emphasis
- **Neutral**: `neutral-{50-900}` - Text, backgrounds, and borders
- **Success**: `success-{100,500,700}` - Positive states and confirmations
- **Warning**: `warning-{100,500,700}` - Caution and attention states
- **Error**: `error-{100,500,700}` - Error states and destructive actions

### Spacing Tokens
- **Scale**: `{0, px, 0.5, 1, 2, 3, 4, 6, 8, 12, 16, 20, 24}` 
- **Usage**: Padding, margins, gaps, and component spacing

### Shadow Tokens
- **Levels**: `{sm, md, lg, xl}` for different elevation levels
- **Usage**: Cards, modals, dropdowns, and interactive elements

## Best Practices

1. **Use semantic tokens first**: Prefer `primary-500` over specific color values
2. **Consistent spacing**: Use the spacing scale for all layout decisions
3. **Appropriate elevation**: Match shadow levels to component hierarchy
4. **Accessible contrast**: Test color combinations for readability
5. **Progressive enhancement**: OKLCH works in modern browsers with automatic fallbacks
