# Wylie Dog Design System

A modern, accessible design system built with React, Radix UI, and OKLCH color science.

## Overview

The Wylie Dog Design System provides a comprehensive set of reusable components, design tokens, and patterns for building consistent, accessible user interfaces. It combines accessible Radix UI primitives with a Figma-synced token pipeline powered by OKLCH color science for perceptually uniform, wide-gamut color across every component.

## Key Features

- **OKLCH Color Science** — Perceptually uniform color space with P3 wide gamut support. Colors look consistent across lightness levels and render beautifully on modern displays.
- **Three-Tier Token Architecture** — Primitives, semantic aliases, and component-level tokens. Synced bidirectionally with Figma via Token Bridge in W3C DTCG format.
- **Accessibility First** — Built on Radix UI primitives with ARIA patterns, keyboard navigation, and focus management baked into every component.
- **44 Components + 5 Compositions** — Production-ready building blocks from inputs and dialogs to full page layouts, all sharing the same token foundation.
- **Modern Stack** — React 19, TypeScript 5.9, Tailwind CSS v4, Vite 7, and Storybook 10.

## Design Tokens

The system is powered by over 1,000 tokens organized in three tiers:

**Primitive tokens** define the raw values — color scales, font sizes, spacing steps:

```css
--color-blue-500: oklch(0.623 0.188 259.82);
--color-gray-900: oklch(0.279 0.037 260.04);
--spacing-4: 1rem;
--font-size-lg: 1.125rem;
```

**Semantic tokens** map primitives to intent, with automatic light/dark mode switching:

```css
/* Light mode */
--color-background-primary: oklch(1 0 0);         /* gray-50 */
--color-text-primary: oklch(0.279 0.037 260.04);   /* gray-900 */

/* Dark mode */
--color-background-primary: oklch(0.208 0.04 265.75);  /* gray-950 */
--color-text-primary: oklch(0.984 0.003 248.29);        /* gray-100 */
```

**Component tokens** tie everything together at the component level:

```css
--color-button-primary-background: var(--color-interactive-primary);
--spacing-button-height-md: var(--spacing-10);
```

Components reference these tokens through Tailwind's CSS variable syntax:

```jsx
// How components use tokens internally
<button className="bg-(--color-button-primary-background) text-(--color-button-primary-text)">
  Save Changes
</button>
```

As a consumer, you don't need to think about token wiring — just use the components:

```jsx
import { Button } from '@wyliedog/ui/button';
import { Input } from '@wyliedog/ui/input';
import { Label } from '@wyliedog/ui/label';

export function LoginForm() {
  return (
    <form className="space-y-4">
      <div>
        <Label htmlFor="email">Email</Label>
        <Input id="email" type="email" placeholder="you@example.com" />
      </div>
      <Button className="w-full">Sign In</Button>
    </form>
  );
}
```
