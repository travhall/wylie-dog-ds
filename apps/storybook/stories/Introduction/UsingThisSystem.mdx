# Using This System

A quick guide to navigating and getting the most from this Storybook.

## Documentation Structure

The sidebar is organized into six sections:

### 1. Introduction
You're here. Setup guides and an overview of the design system.

### 2. Foundations
The building blocks that every component shares:
- **Design Principles** — The reasoning behind our design decisions
- **Colors** — OKLCH color palette with 10 scales, semantic aliases, and interactive token browser
- **Typography** — Type scale, font weights, and font family tokens
- **Spacing & Layout** — Spacing steps and layout tokens
- **Accessibility Guidelines** — Standards for inclusive, WCAG-compliant interfaces

### 3. Components
All 44 UI components, organized by function:
- **Inputs & Controls** — Button, Input, Checkbox, Select, Slider, Switch, and more
- **Navigation** — Tabs, Breadcrumb, Menubar, Command Palette, Pagination
- **Layout & Structure** — Separator, AspectRatio, ScrollArea, Resizable
- **Content Display** — Card, Badge, Avatar, Accordion, Table, Calendar, Carousel
- **Feedback & Status** — Alert, Progress, Skeleton, Toast
- **Overlays & Popovers** — Dialog, Sheet, Popover, Tooltip, DropdownMenu, ContextMenu

### 4. Patterns
Multi-component compositions for common use cases:
- **Form Patterns** — Settings forms, multi-step forms, validation patterns
- **Authentication Patterns** — Login, registration, password recovery flows
- **Data Patterns** — Card grids, feature grids
- **Navigation Patterns** — Site header, site footer
- **Layout Patterns** — Full-page layout compositions
- **Feedback Patterns** — Error boundaries
- **Page Compositions** — Complete page-level examples: full layouts, hero sections, feature grids

### 5. Resources
In-depth implementation guides:
- **Getting Started** — Detailed installation reference and quickstart tutorial
- **Theming** — Light/dark mode, OKLCH color customization, token overrides
- **Guides** — Composition patterns, component testing, performance
- **Contributing** — How to extend and contribute to the system

### 6. Contributing
How to extend and contribute to the system:
- **Guidelines** — Code standards, design standards
- **Workflows** — Component development, design token workflow, release process

## Navigating a Component Page

Each component page provides:

1. **Docs view** — Component description, all story variants rendered inline, and a prop table generated from TypeScript types
2. **Individual stories** — Click any story in the sidebar to view it in isolation on the Canvas
3. **Controls panel** — Interactive prop editor in the bottom panel (toggle with the `Controls` tab)
4. **Accessibility panel** — Run the a11y checker from the `Accessibility` tab (click the play button to scan)

## How to Find What You Need

### Quickest path: Search
Use the search field at the top of the sidebar (`/` shortcut) to jump directly to any component or page.

### For Designers
1. Start with [Design Principles](?path=/docs/foundations-design-principles--docs) to understand the rationale
2. Explore [Colors](?path=/docs/foundations-colors--docs) and [Typography](?path=/docs/foundations-typography--docs) for the visual foundation
3. Browse component stories to see every variant and state

### For Developers
1. Follow the [Getting Started Guide](?path=/docs/introduction-getting-started--docs) for installation
2. Try the [Quickstart Tutorial](?path=/docs/resources-getting-started-quickstart-tutorial--docs) for a hands-on walkthrough
3. Read the [Composition Patterns](?path=/docs/resources-development-composition-patterns--docs) guide for advanced usage

### For Product Teams
1. Review [Patterns](?path=/docs/patterns-overview--docs) for pre-built solutions to common flows
2. Check [Page Compositions](?path=/docs/patterns-page-compositions-page-layout--docs) for reference page layouts
3. Use [Accessibility Guidelines](?path=/docs/foundations-accessibility-guidelines--docs) for compliance requirements

## Toolbar Features

The top toolbar provides:

- **Theme toggle** — Switch between Light, Dark, and System preference modes
- **Viewport selector** — Test components at different screen sizes
- **Canvas / Docs toggle** — Switch between isolated component view and documentation view
