# Using This System

Learn how to navigate and make the most of the Wylie Dog Design System documentation.

## Documentation Structure

The design system is organized into seven main sections, each serving a specific purpose:

### 1. Introduction
Start here if you're new to the system. Learn what it is, how to get started, and how to navigate the documentation.

### 2. Foundations
Explore the core principles and building blocks:
- **Design Principles**: The "why" behind our design decisions
- **Design Tokens**: Visual design primitives (colors, typography, spacing, etc.)
- **Accessibility Guidelines**: Standards and best practices for inclusive design

### 3. Components
Browse reusable UI components organized by function:
- **Inputs & Controls**: Form elements and interactive controls
- **Navigation**: Menus, tabs, breadcrumbs, and navigation patterns
- **Layout & Structure**: Grid systems, containers, and layout primitives
- **Content Display**: Cards, tables, lists, and content presentation
- **Feedback & Status**: Alerts, toasts, progress indicators
- **Overlays & Popovers**: Modals, dialogs, tooltips, and floating elements

### 4. Patterns
Discover multi-component solutions for common use cases:
- **Form Patterns**: Complete form layouts and validation patterns
- **Data Patterns**: Tables, filtering, search, and data loading states
- **Navigation Patterns**: App navigation and responsive menu structures

### 5. Examples
See real-world compositions and application flows:
- **Page Compositions**: Complete page layouts (dashboard, settings, etc.)
- **Common Application Flows**: Multi-step processes and user journeys

### 6. Resources
Task-oriented guides for implementation:
- **Getting Started**: Installation and quick tutorials
- **Theming**: Customizing tokens and creating themes
- **Development**: Component APIs and TypeScript usage
- **Design**: Figma kit and design token workflow
- **Testing & Quality**: Testing and performance monitoring
- **Migration & Updates**: Changelog and upgrade guides

### 7. Contributing
Guidelines for contributing to the design system.

2. Check [Patterns](?path=/docs/patterns-overview--docs) for multi-component solutions

### For Developers

**Getting started:**

## Tips for Navigation

- **Use the search** to quickly find components or topics
- **Browse by category** when exploring
- **Check "Docs" tab** for detailed documentation
- **Try "Canvas" tab** for interactive examples
- **Use the theme toggle** to test dark mode
- **Adjust viewport** to test responsive behavior

## Component Documentation Format

Each component page includes:

1. **Overview**: Purpose and key features
2. **Props**: Interactive controls to explore variations
3. **Usage Guidelines**: Best practices and patterns
4. **Accessibility**: ARIA attributes and keyboard support
5. **Code Examples**: Copy-paste ready implementations
