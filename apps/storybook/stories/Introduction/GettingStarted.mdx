# Getting Started

Get up and running with the Wylie Dog Design System in minutes.

## Installation

Install the UI library and tokens package:

```bash
# Using pnpm (recommended)
pnpm add @wyliedog/ui @wyliedog/tokens

# Using npm
npm install @wyliedog/ui @wyliedog/tokens

# Using yarn
yarn add @wyliedog/ui @wyliedog/tokens
```

## Setup

### 1. Import Styles

Import the design system's stylesheet in your app's entry point. This brings in all component styles and design token CSS custom properties:

```tsx
// app/layout.tsx (Next.js) or main.tsx (Vite)
import '@wyliedog/ui/styles';
```

That single import gives you every token and component style. No Tailwind configuration required on your end unless you want to use Tailwind utilities in your own code.

### 2. TypeScript Configuration

For full type safety, make sure your `tsconfig.json` includes:

```json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "moduleResolution": "bundler",
    "strict": true
  }
}
```

## Your First Component

Components are imported individually from their subpaths:

```tsx
import { Button } from '@wyliedog/ui/button';

export function MyComponent() {
  return (
    <Button onClick={() => alert('Hello!')}>
      Click Me
    </Button>
  );
}
```

The default variant is used when no `variant` prop is specified. Available variants include `default`, `primary`, `secondary`, `outline`, `ghost`, `link`, and `destructive`.

## Building a Form

Here's a complete login form using multiple components:

```tsx
import { Button } from '@wyliedog/ui/button';
import { Input } from '@wyliedog/ui/input';
import { Label } from '@wyliedog/ui/label';
import { Checkbox } from '@wyliedog/ui/checkbox';

export function LoginForm() {
  return (
    <form className="space-y-4 w-full max-w-sm">
      <div className="space-y-2">
        <Label htmlFor="email">Email Address</Label>
        <Input
          id="email"
          type="email"
          placeholder="you@example.com"
          required
        />
      </div>

      <div className="space-y-2">
        <Label htmlFor="password">Password</Label>
        <Input
          id="password"
          type="password"
          placeholder="Enter your password"
          required
        />
      </div>

      <div className="flex items-center gap-2">
        <Checkbox id="remember" />
        <Label htmlFor="remember" className="text-sm">
          Remember me
        </Label>
      </div>

      <Button type="submit" className="w-full">
        Sign In
      </Button>
    </form>
  );
}
```

## Using Design Tokens

Components use design tokens automatically — you get theming and dark mode support without any extra work. If you need to reference tokens directly in your own markup, use the CSS custom property syntax:

```tsx
// Reference semantic tokens via CSS variables
<div style={{ backgroundColor: 'var(--color-background-secondary)' }}>
  Styled with a semantic background token
</div>
```

If your project uses Tailwind CSS v4, you can use the parenthesized variable syntax:

```tsx
// Tailwind v4 CSS variable syntax
<div className="bg-(--color-background-secondary) text-(--color-text-primary) p-(--spacing-4)">
  Styled with tokens through Tailwind
</div>
```

See the [Colors](?path=/docs/foundations-colors--docs) page for the full token browser and three-tier architecture.

## Composing Components

Components are designed to work together:

```tsx
import { Card, CardHeader, CardTitle, CardContent } from '@wyliedog/ui/card';
import { Button } from '@wyliedog/ui/button';
import { Badge } from '@wyliedog/ui/badge';

export function UserCard() {
  return (
    <Card className="w-80">
      <CardHeader>
        <div className="flex items-center justify-between">
          <CardTitle>Jane Doe</CardTitle>
          <Badge variant="success">Active</Badge>
        </div>
      </CardHeader>
      <CardContent className="space-y-4">
        <p className="text-sm text-(--color-text-secondary)">
          Senior Frontend Developer
        </p>
        <div className="flex gap-2">
          <Button size="sm">Message</Button>
          <Button variant="outline" size="sm">View Profile</Button>
        </div>
      </CardContent>
    </Card>
  );
}
```

## Handling States

Components support contextual variants for feedback:

```tsx
import { Alert, AlertTitle, AlertDescription } from '@wyliedog/ui/alert';

export function Notifications() {
  return (
    <div className="space-y-4">
      <Alert variant="success">
        <AlertTitle>Success!</AlertTitle>
        <AlertDescription>Your changes have been saved.</AlertDescription>
      </Alert>

      <Alert variant="warning">
        <AlertTitle>Warning</AlertTitle>
        <AlertDescription>This action cannot be undone.</AlertDescription>
      </Alert>

      <Alert variant="destructive">
        <AlertTitle>Error</AlertTitle>
        <AlertDescription>Something went wrong. Please try again.</AlertDescription>
      </Alert>
    </div>
  );
}
```

## Accessibility

All components are built on [Radix UI](https://www.radix-ui.com/) primitives with full keyboard navigation, focus management, and ARIA attributes. Follow these patterns for best results:

```tsx
import { Button } from '@wyliedog/ui/button';
import { Input } from '@wyliedog/ui/input';
import { Label } from '@wyliedog/ui/label';

export function AccessibleForm() {
  return (
    <form>
      {/* Always pair inputs with labels */}
      <Label htmlFor="username">Username</Label>
      <Input
        id="username"
        aria-describedby="username-help"
        required
      />
      <p id="username-help" className="text-sm text-(--color-text-tertiary)">
        Choose a unique username
      </p>

      <Button type="submit">Submit</Button>
    </form>
  );
}
```

## Responsive Design

Components work at any viewport width. Use Tailwind's responsive utilities for layout:

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <Card>Stacks on mobile</Card>
  <Card>Two columns on tablet</Card>
  <Card>Three columns on desktop</Card>
</div>
```
